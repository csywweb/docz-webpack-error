## 简介

如果业务中直接使用zent提供的Table组件，每一个业务都需要去维护datasets、当前页码、监听翻页动作、调用接口拉取数据等等操作，使用复杂度较高。
在美业业务里面，接口结构基本统一，很多参数都是固定的，鉴于此，在zent的Table组件基础之上，封装了beauty_table组件，用户只需要配置基本的参数，
就可以使用Table的所有功能，无需再监听翻页事件、维护datasets及当前页码等繁琐的操作。

> owner 思远

## 引入

```js
import BeautyTable from '@youzan/beauty-react-components';
```

## 参数

| 参数 | 类型 | 是否必填        |默认值   |说明
|------|------------------|-----------------|---------|----------|
|pageInfo|object|否|{ pageNo: 1, pageSize: 20 }|表格翻页翻页参数，仅pageNo和pageSize两个字段
|params|object|否|{}|接口调用参数，比如deptId
|fetchData|Promise|是||翻页api接口，必须是Promise对象
|formatData|func|否|noop|格式化数据函数，如果接口返回的数据不是标准格式，需要通过该函数序列化成标准格式
|columns|array|是|[]|表格列
|autoFetchData|boolean|否|true|是否主动渲染数据
|onChange|function|否|noop|Table的onChange方法，参数 sortType, sortBy 表格带排序时使用
|onTotalChange|function|否|noop|加载完成的函数回调，参数 total, 总条数
|onFetchDataError|function|否|noop|加载错误的函数回调，参数 err
|...|...|否|undefined|zent的Table组件其他配置都可以使用，参考[Table](https://youzan.github.io/zent/zh/component/table)

## 外部调用方法
### refreshTable
在组件外部，可以通过ref调用表格的refreshTable方法，刷新当前页面，eg：
```
this.table.refreshTable()
```
### resetTable  
在组件外部，通过ref调用表格的resetTable，把表格重置到首页并刷新，eg：
```
this.table.resetTable()
```
